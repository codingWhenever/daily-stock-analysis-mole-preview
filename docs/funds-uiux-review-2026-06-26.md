# 选基模块产品 + UI/UX 走查与工程改造单

更新时间：2026-06-26
评审人视角：金融产品经理 + 高级 UI/UX 设计师
评审对象：`/funds` 选基决策工作台（前端 `apps/dsa-web/src/pages/FundsPage.tsx` 及其子组件）

> 本文是「拿来即改」的工程文档。每个问题都给出：**现象 → 影响 → 证据 → 根因 → 改法（具体文件/组件/行号）→ 验收**。
> 配套定位与实现边界以 `docs/funds-product-review-2026-06-26.md`、`docs/funds-handoff-2026-06-26.md` 为准，本文不重复其结论，只在其基础上补「界面层」的诊断与落地动作。

## 0. 一句话结论

选基模块的**产品骨架与数据契约已经相当完整**（市场榜单 / 荐基证据 / 持仓导入 / 个人动作 / 基金池画像五层都已落地第一版），但**界面层把"一个长滚动页面"当成了信息架构**：桌面端整页约 **13,300px（≈11 屏）**，移动端约 **18,800px（≈23 屏）**，且移动端存在明确横向溢出。当前界面的最大问题不是缺功能，而是**所有模块平铺、负面/空值信息主导、缺少分区与状态视觉分层**，导致"决策工作台"读起来像"调试面板"。

本轮工程重心建议：**先做信息架构收口（分区 + 折叠 + 移动端不溢出），再做证据卡/持仓/个人动作三块的"空态降级"与"状态视觉分层"**。不需要新增数据能力即可显著提升可用性。

## 2. 当前信息架构现状（实测渲染顺序）

`FundsPage` 的 `return` 是一条自上而下的平铺流（`apps/dsa-web/src/pages/FundsPage.tsx:3168`）：

| 顺序 | 区块 | 组件 / 入口 | 实测纵向占用 | 备注 |
| --- | --- | --- | --- | --- |
| 1 | 页头 + 刷新基金池 | `PageHeader` (`:3170`) | 小 | — |
| 2 | 市场公开榜单 + 今日荐基 | `MarketRankingWorkbench` (`:1369` / `:3188`) | **极大（≈0–4,000px）** | 榜单表 + 10 张荐基证据卡 |
| 3 | 「决策分层」提示条 | `InlineAlert` (`:3206`) | 小 | 离它说明的"持仓/个人"对象太远 |
| 4 | 搜基入口 + 搜索结果 | `:3212` / `FundSearchCard` (`:2067`) | 中 | 已知代码用户被迫滚过榜单才到这里 |
| 5 | 3 个 StatCard | `:3280` | 小 | 当前值多为 0/`--` |
| 6 | 持仓导入助手 + 已确认持仓 | `FundHoldingImportAssistant`（独立文件） | **极大（≈4,200–7,900px）** | 聚合卡 + 全量明细表**双重渲染** |
| 7 | 个人持仓动作 | `PersonalActionsPanel` (`:1660` / `:3311`) | **大（≈8,000–11,700px）** | 当前 100% 阻塞 |
| 8 | 账本切换 + 账本画像 | `FundLedgerSwitcher` (`:1854` / `:3318`) | 中 | — |
| 9 | 基金池 master-detail | `FundPoolWorkspace` (`:2638` / `:3350`) | 大（内含列表内滚动） | 选中单品才展开详情 |

**结论**：真正"我的资产/要不要动作"（持仓 + 个人动作 + 基金池）全部沉在 4,000px 以下，而首屏被"市场级公开榜单"长内容占满。这与产品文档自己定义的优先级（"第一屏能回答：今天市场值得看什么 + 我的持仓是否需要动作"）相悖。

## 3. 问题清单（按严重级排序）

严重级定义：**P0 = 破坏可用性，必须改**；**P1 = 严重损害理解/信任**；**P2 = 一致性/打磨**。

### P0-1　整页纵向超长、无分区导航

- **现象**：桌面 ≈11 屏、移动 ≈23 屏的单一滚动流，无 tab、无锚点、无 sticky 区块导航。
- **影响**：用户无法快速跳到"我的持仓"或"基金池"；每次操作后回到原位成本极高；移动端尤甚。
- **证据**：`scrollHeight` 13,303 / 18,769；`FundsPage` return 为线性 `space-y-5` 堆叠（`:3169`）。
- **根因**：把"模块齐全"等同于"全部同屏展示"。
- **改法**（二选一，推荐 A）：
  - **A（推荐，改动可控）**：在 `FundsPage` 顶部加一条 **sticky 分区导航**，把现有区块归并为 4 个语义分组并支持锚点跳转 + 默认折叠非首屏分组：
    1. **今日市场**（区块 2）
    2. **搜基 / 基金池**（区块 4、9）
    3. **我的持仓**（区块 6、7）
    4. **账本与画像**（区块 8）
    实现：新增 `FundsSectionNav`（sticky top，`position: sticky; top: 0`），各区块包一层带 `id` 的 `<section>`；非首屏分组用受控折叠（`useState` + `Collapsible`），首次进入只展开"今日市场 + 搜基/基金池"。
  - **B（更彻底）**：把 `/funds` 拆为子路由 tab（`/funds`、`/funds/holdings`、`/funds/pool`），各 tab 独立数据加载，避免一次性拉全部接口。
- **验收**：首屏（不滚动）即可看到分区导航；任意分组一次点击可达；移动端整页可达内容数量不变但首屏高度 < 2 屏。

### P0-2　移动端横向溢出 + 悬浮按钮遮挡

- **现象**（375px 实测）：
  1. 已确认持仓**明细表是 9 列固定像素网格**（`grid-cols-[82px_…_76px]` 合计 ≈870px），在 375px 下严重横向溢出/截断。
  2. 个人动作卡片右侧 badge（如 `先屏…`、`回测 missing…`）被裁切。
  3. 右上角悬浮「界面语言 / 深色」按钮**压在内容之上**，遮挡"单品分析数据质量未达 ok"等文案。
- **影响**：移动端关键数据读不全，观感失控。
- **证据**：`FundHoldingImportAssistant.tsx:586` 与 `:603` 的固定列宽表头/表体；移动截图可见截断与遮挡。
- **根因**：明细表用桌面固定列宽网格，未做移动端降级；悬浮工具条无内容避让。
- **改法**：
  - 明细表：移动端（`< sm`）改为**每行卡片**或包一层 `overflow-x-auto` 且显式给出"左右滑动查看"提示；或直接复用聚合卡布局（见 P0-3 合并方案后此表可删）。文件：`FundHoldingImportAssistant.tsx:586-630`。
  - 悬浮工具条：给主内容容器加 `padding-top` 或把语言/主题按钮在窄屏移入抽屉/汉堡菜单（定位组件在 `ShellHeader.tsx`）。
- **验收**：375px 下无横向滚动条（除非显式横滑容器）；无任何文字被悬浮层遮挡。

### P0-3　持仓被双重渲染（聚合卡 + 全量明细表）

- **现象**：同一份已确认持仓，先以"聚合卡"（最多 8 张）渲染一遍，紧接着又以"全量明细表"（31 行 × 9 列）渲染一遍。
- **影响**：信息冗余、撑高页面（持仓段独占约 3,700px），且两种视图字段口径不同易混淆。
- **证据**：`FundHoldingImportAssistant.tsx:524-545`（聚合卡，`aggregatedHoldings.slice(0,8)`）与 `:586-630`（明细表全量）。
- **根因**：聚合视图与平台明细未做"同一对象的两种视图切换"，而是上下并列。
- **改法**：合并为**单一列表 + 视图切换**：默认显示"按基金聚合"卡片（去掉 `.slice(0,8)` 截断或改为"展开全部"），明细表收进每张卡的"展开/平台明细"折叠区，或顶部加 `聚合 / 明细` 二选一切换（`Segmented`）。文件：`FundHoldingImportAssistant.tsx:495-630`。
- **验收**：持仓默认只占 1 种视图高度；用户可主动切到明细；持仓段总高 < 原来一半。

### P1-1　荐基证据卡：负面/失效信息主导，单卡过高

- **现象**：今日荐基 10 张卡，每张 ≈350px，**每张都重复**展示 `净值样本不足 / NAV 样本 0`、`待分析 not_in_pool`、`未生成单品画像`、`回测样本不足`，外加整段「失效条件：公开榜单数据日期明显滞后…」长文案。
- **影响**：本应"推荐你看什么"的模块，读起来全是"什么都还没准备好"，正向证据（净流入、热度、收益排名）被淹没；10 张卡进一步拉长页面。
- **证据**：`RecommendationEvidenceCard` (`:1245`)；实测每卡含 2~3 个风险 badge + 失效条件段落。
- **根因**：把"数据可信边界"（值得保留）做成了"每卡全量平铺"（不该默认全展开）。
- **改法**：
  - 卡片**默认只展示**：排名、名称/代码、SCORE、1~2 条**正向证据**（买入/热度/收益）、一个总体状态 chip（如 `仅市场级证据`）。
  - 把"回测准备 / 单品画像 / 风险 badge / 失效条件"收进**"查看证据与边界"折叠**（默认收起）。
  - 失效条件文案在同组卡片完全相同时，**上提为该榜单的一条统一说明**，不在每卡重复。
  - 文件：`FundsPage.tsx:1245-1368`（卡片结构）、`:1369-1659`（workbench 容器，放统一说明）。
- **验收**：单卡默认高度 < 160px；正向证据在折叠前可见；失效条件不再逐卡重复。

### P1-2　个人动作区：100% 阻塞态仍逐卡平铺

- **现象**：当前「动作 0 / 阻塞 3」，每张个人动作卡都显示"单品分析数据质量未达 ok，暂不输出金额区间 / 缺少生成个人动作所需的分析或画像"，但该区仍占约 3,600px。
- **影响**：用户面对一长串"无法给建议"的卡片，认知负担高且无行动指引；与产品文档验收"无画像或无已确认持仓时不输出个人动作"精神不符。
- **证据**：`PersonalActionsPanel` (`:1660`)；实测顶部 stat「动作 0 / 阻塞 3」，下方仍渲染逐条阻塞卡。
- **根因**：未对"整体未达可输出条件"做**区块级降级**，而是下沉到每卡显示空值。
- **改法**：当"可输出动作数 = 0"时，**整段折叠为单一引导态**：一句话说明为什么未启用（缺单品分析 / 缺画像 / 缺现金输入）+ 一个明确的下一步 CTA（"先刷新单品分析"/"完善账本画像"）；详细逐卡阻塞放进"查看 N 条阻塞明细"折叠。文件：`FundsPage.tsx:1660-1853`。
- **验收**：无可输出动作时该区高度 < 1 屏；有可输出动作时才逐卡展开；引导态给出可点击的下一步。

### P1-3　闭眼模式信息空洞

- **现象**：金额闭眼开启时，31 张持仓卡的「市值/成本/盈亏/收益率/成本单价/份额」**6 个字段全部 `••••`**，每卡仅"名称 + 代码 + 最新净值"可读。
- **影响**：闭眼态下持仓段几乎不承载任何可读信息，却占满纵向空间。
- **证据**：`FundHoldingImportAssistant.tsx:39-66`（`maskedNumber` → `••••`），聚合卡/明细表均走遮蔽。
- **根因**：遮蔽策略是"逐字段打码"，没有"闭眼态专用紧凑布局"。
- **改法**：闭眼时切换为**紧凑单行**（名称 + 代码 + 净值 +「盈亏方向」用纯色箭头/涨跌色块表示但不显示数值），并提供"按占比"的可视条（占比是相对值、不泄露金额）。文件：`FundHoldingImportAssistant.tsx:524-630`。
- **验收**：闭眼态下每条持仓高度显著下降；仍能看出"涨/跌方向"和"相对占比"而不暴露金额。

### P1-4　密集徽章 / 评分无图例

- **现象**：`代理口径`、`回测 missing`、`市场上下文 62`、`仓位约束 76`、`用户画像 100`、`单品分析 45`、`总分 66.3` 等大量徽章/分值同屏出现，**无统一图例与口径说明**。
- **影响**：非专业用户无法理解分值含义与权重；专业用户也需猜测口径。
- **证据**：个人动作卡与荐基卡均密集挂分值 badge（`PersonalActionsPanel` / `RecommendationEvidenceCard`）。
- **根因**：评分子项直出，缺解释层。
- **改法**：
  - 在个人动作/荐基区顶部加一个**统一图例**（hover/点击展开）：解释"总分 = 各子项加权"、各子项口径（市场上下文 / 仓位约束 / 用户画像 / 单品分析）。
  - 子项 badge 加 `title`/tooltip。文件：`FundsPage.tsx:1245-1853` 内复用一个 `ScoreLegend` 小组件。
- **验收**：任一分值 hover 有解释；区块顶部有一处图例可达。

### P2-1　四态数据状态缺乏视觉分层

- **现象**：产品契约要求标注 `已实接 / 计算中 / 待接入 / 需用户确认`，但界面上 `ok / partial / missing / 代理口径 / 待分析 / 未知` 等都渲染成**同一类小徽章**，颜色区分弱。
- **影响**：用户/工程都难一眼区分"真实数据"与"占位/代理"，违背文档强调的"不把缺口显示成已接入"。
- **改法**：定义一套**状态视觉令牌**（4 态固定语义色 + 图标）：真实直出（success/实心）、计算中（info/进度感）、待接入（neutral/虚线）、需用户确认（warning/描边）。集中在现有 `coverageVariant/contextVariant/evidenceVariant`（`FundsPage.tsx:454-487`）之上再抽一层"语义态 → 视觉令牌"映射，全页统一调用。
- **验收**：同一态在榜单/荐基/画像/数据覆盖处视觉一致；"代理/估算"与"真实直出"一眼可辨。

### P2-2　搜基入口与「决策分层」提示位置

- **现象**：搜基入口在长榜单之后（区块 4）；「决策分层」`InlineAlert` 在榜单与搜索之间，但它解释的"持仓/个人动作"在 7,000px 以下。
- **改法**：搜基入口上移至页头下方常驻（已知代码用户高频）；「决策分层」提示改为各分区内的轻量副标题，而非独立大横条。文件：`FundsPage.tsx:3206-3244`。
- **验收**：进入页面 1 屏内可发起搜索；提示文案紧邻其所描述的对象。

### P2-3　StatCard 与重复文案

- **现象**：3 个 StatCard（基金池 / 可执行信号 / 数据可用）当前多为 `0/--`，与下方账本切换的计数语义重复；多处"失效条件""数据口径"长文案逐卡复制。
- **改法**：StatCard 合并进分区头部的统计条；重复长文案上提为区块级一次性说明。文件：`FundsPage.tsx:3280-3302`。
- **验收**：同一信息全页只出现一次；空值不再单独占大卡。

## 4. 推荐目标信息架构（设计稿口径，供工程对齐）

```
/funds 选基决策工作台
├─ 顶部：页头 + 常驻搜基入口 + sticky 分区导航（4 锚点）
│
├─ ① 今日市场（默认展开）
│    ├─ 市场公开榜单（榜单 tab：流入/流出/热度/收益实证）—— 表格保留
│    └─ 今日荐基候选 —— 卡片默认精简，证据/边界折叠，失效条件统一说明
│
├─ ② 搜基 / 基金池（默认展开）
│    ├─ 搜索结果
│    └─ 基金池列表（内滚动）+ 选中单品详情抽屉（画像/回测/信号/同类）
│
├─ ③ 我的持仓（默认折叠，有持仓时展开）
│    ├─ 持仓导入助手
│    ├─ 已确认持仓（单一视图 + 聚合/明细切换；闭眼态紧凑）
│    └─ 个人持仓动作（0 动作时整体降级为引导态）
│
└─ ④ 账本与画像（默认折叠）
     └─ 账本切换 + 账本画像问卷
```

关键设计原则（与产品文档一致，落到界面）：
1. **第一屏回答两件事且不混淆**：今天市场看什么（市场级）/ 我的持仓要不要动（个人级）。
2. **每块数据显式标注四态**（见 P2-1）；缺口绝不伪装成已接入。
3. **空态 = 引导态**：没有就给下一步，而不是平铺一堆 `--`。
4. **证据可信边界保留但默认收起**：让正向信号先被看到。
